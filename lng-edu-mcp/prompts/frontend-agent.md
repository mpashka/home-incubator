# Задание агенту: Vue web-клиент

@tag:vertical-slice

Работай в репозитории `lng-edu-mcp`. Сначала прочитай `AGENTS.md`, `README.md`, `docs/architecture.md`, актуальную OpenAPI-схему backend и его инструкции запуска.

Создай в `web/` Vue 3 приложение на TypeScript strict и Vite. Используй Vue Router и Pinia. Цель — рабочий интерфейс первого вертикального учебного сценария, а не демонстрационный набор экранов.

## Экраны и сценарии

1. Выбор профиля и языка.
2. Выбор книги с прогрессом чтения.
3. Учебная сессия:
   - текущий фрагмент;
   - действия «Понятно», «Частично», «Непонятно»;
   - ввод незнакомого слова;
   - получение следующего фрагмента;
   - завершение сессии.
4. Словарь: слово, язык, уровень/статус, последний контекст и время встречи.
5. Статистика: минуты сегодня, серия дней, прочитанные фрагменты и новые слова.

## Требования

- Используй Composition API и небольшие компоненты.
- Централизуй API client и типизированные DTO; если OpenAPI стабилен, генерируй типы.
- Показывай loading, empty и error states.
- Не выполняй optimistic updates там, где можно потерять учебное событие.
- Интерфейс должен работать на телефоне и desktop, иметь видимый focus и корректные labels.
- Тексты UI подготовь на русском; архитектуру сделай пригодной для последующей локализации.
- Добавь unit/component tests для основной логики и mock API.
- Настрой ESLint, formatter, typecheck и production build.
- Добавь проксирование `/api` для локальной разработки и документируй переменные окружения.

## Критерии приемки

- `npm run lint`, `npm run typecheck`, `npm test -- --run` и `npm run build` проходят;
- пользователь может пройти сценарий profile → book → session → result/word → finish → stats;
- обновление страницы не создает повторное учебное событие;
- ошибки backend понятны пользователю и не теряются молча;
- README содержит точные команды запуска frontend с backend.

Не добавляй microphone/audio, собственный чат, SSR и сложную дизайн-систему на первом этапе.

В конце сообщи: что реализовано, команды проверки, использованные API endpoints и оставшиеся ограничения.
