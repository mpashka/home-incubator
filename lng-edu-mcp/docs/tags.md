# Теги (llm-wiki-plus)

Реестр тегов, связывающих сквозные концепции между кодом и документацией. Каждый тег регистрируется здесь один раз с однострочным описанием.

Родитель: [`index.md`](index.md).

## Реестр

| Тег | Концепция |
| --- | --- |
| `@tag:vertical-slice` | Первый вертикальный учебный сценарий: start session → next chunk → result/unknown word → finish → stats. |
| `@tag:mcp-tools` | Набор MCP tools MVP и их контракты для LLM-клиентов. |
| `@tag:request-id` | Идемпотентность изменяющих операций: повтор с тем же `request_id` не создаёт дубль. |
| `@tag:domain-model` | Начальная модель данных и таблицы (users, user_language_skills, books, book_texts, reading_progress, learning_sessions, vocabulary_items, word_events). |
| `@tag:reading-block` | Динамический блок чтения и его размер как свойство связки «пользователь + язык»; позиция — символьное смещение. |
| `@tag:difficulty-model` | Будущее: оценка сложности слов и уровня пользователя для подбора доли незнакомых слов в блоке. |

На текущем этапе исходный код backend/web ещё не создан, поэтому теги размещены в документации. При написании соответствующего кода добавляй тот же `@tag:<slug>` в комментарий у начала файла (см. соглашение в [`../AGENTS.md`](../AGENTS.md)).

## Формат тегов

Документация (`.md`): строка `@tag:<slug>` у начала файла; для каталога — в его `index.md`.

Код: комментарий у начала файла в синтаксисе языка, содержащий токен, например `// @tag:<slug>`, `# @tag:<slug>`, `/* @tag:<slug> */`.

Несколько тегов: повторяй токен через пробел: `@tag:ui @tag:mechanism`.

Slug — строчный kebab-case: `[a-z0-9-]+`.

## Поиск

```bash
# все места в коде и документации с данным тегом
grep -rn "@tag:<slug>" .

# теги конкретного файла
grep -oE "@tag:[a-z0-9-]+" path/to/file

# все теги, используемые в репозитории
grep -rhoE "@tag:[a-z0-9-]+" . | sort -u
```
